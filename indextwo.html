Comparing Windows, Mac and Linux Operating Systems

Windows

Files and Directories, Naming Conventions, and the File Allocation Table (FAT)

This system uses a backslash to clarify file-paths.
For example, C:\User\Folder1\SubFolder\example.exe.

Windows naming conversion is not case-sensitive. So a file names WINDOWS=Windows

There has to be a maximum of 260 characters, containing special characters question mark ?, backslash\, forward-slash /, asterisk *, colon :, quotation mark “, less/greater than < >, and vertical bar |.

Windows 10 and 11 support the File Allocation Table filing system. However, they use the New Technology Filing System (NTFS) by default, and Resilient File System (Reefs) to a much lesser extent.

In Windows, FAT is mostly used by the older versions of the system. Hence from 10 to the current version (i.e., Windows 11), the NTFS is used even though both versions still support the FAT system.

Executable File Extensions

.bt (batch),
.exe(executable),
.msi(Windows installer package),
.cmd(command prompt),
.wsf(Windows script files),
.com(Microsoft-DOS).

C:\Windows\system32\>cmd.exe and enter to run commands.

Windows has shell access, running both on the PoweShell and Terminal

Mac

Files and Directories, Naming Conventions, and the File Allocation Table (FAT)

As opposed to Windows backslash, MacOS uses forward-slashes(/) to show a directory.

For a MacOS, Foldername is not the same as FolderName. Therefore, it is essential to note the cases

MacOS doesn't allow a colon (:) but does for most special characters.

MacOS’s file system uses File Allocation Table (FAT) structure like Windows. However, its default system is an Apple Filing System (APFS) from the MacOS version 10.13 to the latest/current version.

FAT32 is currently being used by the MacOS, it is from the evolution of the FATs original system.

Executable File Extensions

.app(application bundle),
.command(shell scripts),
.dylib(dynamic library),
.dmg(disk image)

As mentioned in the directory structure that MacOS uses forward-slashes, so its script file is /. For example, /User/Folder1/Subfolder/example.app

Mac's Operating System has shell access using the Terminal only

Linux

Files and Directories, Naming Conventions, and the File Allocation Table (FAT)

Almost similar to the MacOS, Linux uses forward slashes(/) for the root directory.

Linux is also case-sensive, so this OS also doesn't allow File and FILE to be equal/same.

There are restricted characters like the forward-slash (/), and the null character, otherwise they allow most characters.

Linux supports FAT32, and NTFS too.

Mostly for compatibility and removable disks, in order to work well with other systems.

Executable File Extensions

Regular files,
directories,
and special files.

Linux can execute from a command line with ./ prefix.

Terminal, and shell access.
